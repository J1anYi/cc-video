# Milestone v1.4: Account Enhancement

**Status:** SHIPPED 2026-04-29
**Phases:** 11-12
**Total Plans:** 3

## Overview

v1.4 focused on account management improvements: password reset via email and user profile management.

## Phases

### Phase 11: Password Reset

**Goal**: Enable users to recover access to their accounts via email-based password reset
**Depends on**: Phase 4 (authentication)
**Plans**: 2 plans

Plans:

- [x] 11-01: Password Reset Backend
- [x] 11-02: Password Reset Frontend

**Details:**
Implemented email-based password reset with secure tokens:
- PasswordReset model with token_hash, expires_at, used_at fields
- Email service with SMTP support and dev mode logging
- Frontend pages (ForgotPassword, ResetPassword)
- 1-hour token expiration, single-use tokens
- No email enumeration for security

### Phase 12: Profile Management

**Goal**: Allow users to view and manage their profile information
**Depends on**: Phase 11
**Plans**: 1 plan

Plans:

- [x] 12-01: Profile Management

**Details:**
User profile viewing and management:
- Added display_name field to User model
- Profile viewing and display name updates
- Password change when logged in
- Profile frontend page

---

## Key Decisions

- Decision: Use SHA-256 hash for reset tokens (Rationale: Secure, no plaintext tokens stored)
- Decision: 1-hour token expiration (Rationale: Balance security and usability)
- Decision: No email enumeration (Rationale: Prevent account discovery attacks)
