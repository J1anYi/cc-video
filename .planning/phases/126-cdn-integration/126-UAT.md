# UAT: Phase 126 - CDN Integration

## Test Cases

### 1. CDN Configuration

#### TC-CDN-001: Create CDN Configuration
**Steps:**
1. Send POST request to `/cdn/configure` with provider, origin_url
2. Verify response includes id, tenant_id, provider, is_active

**Expected:** Configuration created with default TTL settings

#### TC-CDN-002: Update CDN Configuration
**Steps:**
1. Send POST request to `/cdn/configure` with updated settings
2. Verify existing configuration is updated

**Expected:** Configuration updated, same ID returned

#### TC-CDN-003: Get CDN Configuration
**Steps:**
1. Send GET request to `/cdn/config`
2. Verify active configuration returned

**Expected:** Current configuration returned

### 2. Cache Rules

#### TC-CDN-004: Create Cache Rule
**Steps:**
1. Send POST to `/cdn/cache-rules` with path_pattern, behavior, ttl
2. Verify rule created with id

**Expected:** Cache rule created

#### TC-CDN-005: List Cache Rules
**Steps:**
1. Create multiple cache rules
2. Send GET to `/cdn/cache-rules`
3. Verify all active rules returned

**Expected:** Rules returned in priority order

#### TC-CDN-006: Delete Cache Rule
**Steps:**
1. Send DELETE to `/cdn/cache-rules/{rule_id}`
2. Verify success response
3. List rules, verify deleted rule not returned

**Expected:** Rule soft-deleted

### 3. Cache Invalidation

#### TC-CDN-007: Create Invalidation
**Steps:**
1. Send POST to `/cdn/invalidate` with paths list
2. Verify invalidation_id returned
3. Verify status is COMPLETED

**Expected:** Invalidation created and completed

#### TC-CDN-008: Get Invalidation Status
**Steps:**
1. Create invalidation
2. Send GET to `/cdn/invalidate/{invalidation_id}`
3. Verify status returned

**Expected:** Invalidation status returned

### 4. Metrics

#### TC-CDN-009: Record Metrics
**Steps:**
1. Record CDN metrics via service method
2. Query metrics endpoint

**Expected:** Metrics stored and retrievable

#### TC-CDN-010: Get Metrics Summary
**Steps:**
1. Record multiple metrics
2. Send GET to `/cdn/metrics/summary`
3. Verify aggregated totals returned

**Expected:** Summary with totals and hit_rate

## Test Execution

```bash
# Run backend tests
cd backend
pytest tests/test_cdn.py -v

# Manual API testing
curl -X POST http://localhost:8000/cdn/configure \
  -H "Content-Type: application/json" \
  -d '{"provider": "cloudfront", "origin_url": "https://origin.example.com"}'

curl http://localhost:8000/cdn/config

curl -X POST http://localhost:8000/cdn/cache-rules \
  -H "Content-Type: application/json" \
  -d '{"path_pattern": "/videos/*", "ttl": 3600}'

curl http://localhost:8000/cdn/cache-rules

curl -X POST http://localhost:8000/cdn/invalidate \
  -H "Content-Type: application/json" \
  -d '{"paths": ["/videos/test.mp4"]}'

curl http://localhost:8000/cdn/metrics/summary
```
