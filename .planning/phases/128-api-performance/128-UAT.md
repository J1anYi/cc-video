# UAT: Phase 128 - API Performance

## Test Cases

### 1. Response Caching

#### TC-APIP-001: Cache HIT
**Steps:**
1. Send GET request to cacheable endpoint
2. Verify X-Cache: MISS header
3. Send same request again
4. Verify X-Cache: HIT header

**Expected:** Second request served from cache

### 2. Batch Endpoints

#### TC-APIP-002: Batch Execute
**Steps:**
1. Send POST to /batch/execute with multiple operations
2. Verify results for each operation

**Expected:** All operations executed

#### TC-APIP-003: Batch Limit
**Steps:**
1. Send batch with > max_operations
2. Verify error response

**Expected:** 400 error with limit message

### 3. Compression

#### TC-APIP-004: GZip Compression
**Steps:**
1. Send request with Accept-Encoding: gzip
2. Check response Content-Encoding header

**Expected:** Response compressed for large responses

### 4. API Metrics

#### TC-APIP-005: Cache Metrics
**Steps:**
1. Make several cached requests
2. Send GET to /api-metrics/cache
3. Verify hit_rate

**Expected:** Cache statistics returned

#### TC-APIP-006: Performance Summary
**Steps:**
1. Send GET to /api-metrics/summary
2. Verify cache and query stats

**Expected:** Aggregated metrics returned

## Test Execution

```bash
# Test caching
curl http://localhost:8000/health

# Batch operations
curl -X POST http://localhost:8000/batch/execute \
  -H "Content-Type: application/json" \
  -d '{"operations": [{"type": "echo", "params": {"test": 1}}]}'

# API metrics
curl http://localhost:8000/api-metrics/summary

# Clear cache
curl -X POST http://localhost:8000/api-metrics/cache/clear
```
