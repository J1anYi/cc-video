<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> Admin API | Untrusted input crosses here; all admin endpoints require authentication and admin role |
| Admin API -> File System | Files uploaded by admin are stored in controlled directory |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-07 | Tampering | Video upload | mitigate | Validate MIME type against allowed list; validate file size; use UUID filenames to prevent path traversal |
| T-02-08 | Information Disclosure | Video files | accept | v1 stores files in non-public directory; Phase 3 implements authenticated video serving |
| T-02-09 | Denial of Service | Upload endpoint | mitigate | File size limit enforced (MAX_VIDEO_SIZE); timeout on upload operations |
| T-02-10 | Spoofing | Video metadata | mitigate | MIME type from client is validated; filename stored but not used for serving |

## Additional Security Considerations

1. **File Storage Security:**
   - Files stored with UUID-prefixed names (prevents directory traversal attacks)
   - Original filename preserved in database for display only
   - Upload directory outside web root (not directly accessible)

2. **Upload Validation:**
   - MIME type whitelist enforced (only video/* types)
   - File size limit enforced before writing to disk
   - Validation errors return clear HTTP 400 responses

3. **Future Considerations (Phase 3):**
   - Video files served through authenticated endpoint
   - Range request support for seeking
   - Consider virus scanning for production deployment
</threat_model>
