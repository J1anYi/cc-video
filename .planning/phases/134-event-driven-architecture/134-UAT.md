# UAT: Phase 134 - Event-Driven Architecture

## Test Cases

### 1. Event Bus

#### TC-EDA-001: Publish Event
**Steps:**
1. POST to /events/publish with event data
2. Verify event published

**Expected:** Event published successfully

#### TC-EDA-002: Subscribe Handler
**Steps:**
1. Register handler for event type
2. Publish matching event
3. Verify handler called

**Expected:** Handler executed

### 2. Event Store

#### TC-EDA-003: Store Event
**Steps:**
1. Publish event
2. GET /events/store
3. Verify event stored

**Expected:** Event in store

#### TC-EDA-004: Get Aggregate Events
**Steps:**
1. Publish events for aggregate
2. GET /events/store/{aggregate_id}
3. Verify all events returned

**Expected:** All aggregate events returned

### 3. Saga

#### TC-EDA-005: Create Saga
**Steps:**
1. Create saga with steps
2. Execute saga
3. Verify completed

**Expected:** Saga completed

#### TC-EDA-006: Saga Compensation
**Steps:**
1. Create saga with failing step
2. Execute saga
3. Verify compensation ran

**Expected:** Compensating actions executed

## Test Execution

```bash
# Publish event
curl -X POST http://localhost:8000/events/publish \
  -H "Content-Type: application/json" \
  -d '{"event_type": "user.created", "aggregate_id": "user-1", "aggregate_type": "user", "data": {"name": "test"}}'

# List events
curl http://localhost:8000/events/store

# Get dead letter queue
curl http://localhost:8000/events/dead-letter
```
