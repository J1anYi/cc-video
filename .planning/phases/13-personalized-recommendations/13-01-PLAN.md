# Phase 13-01: Recommendations Backend

<objective>
Create backend API endpoints for personalized recommendations and continue watching functionality.
</objective>

- backend/app/models/movie.py — Movie model with category
- backend/app/services/history.py — History service patterns
- backend/app/routes/movies.py — Existing route patterns
</read_first>

<tasks>

<task id="1" type="execute">
<name>Create recommendation schemas</name>
<action>
Create `backend/app/schemas/recommendation.py` with:
- `RecommendedMovie` schema with movie data and recommendation reason
- `ContinueWatchingItem` schema with movie data, progress, and resume position
- `RecommendationsResponse` schema with both lists
</action>
<acceptance_criteria>

</acceptance_criteria>
</task>

<task id="2" type="execute">
<name>Create recommendation service</name>
<action>
Create `backend/app/services/recommendation.py` with:

1. `get_personalized_recommendations(db, user_id, limit=10)`:
   - Get categories from user's watched movies (weighted by recency)
   - Get categories from user's favorites
   - Find movies in those categories, excluding already watched
   - Return top N movies with recommendation reason

2. `get_continue_watching(db, user_id, limit=10)`:
   - Query WatchHistory where user_id matches and progress < 100
   - Order by last_watched_at desc
   - Return movie with progress percentage

3. Simple scoring: favorite categories get 2x weight, watched categories 1x weight
</action>
<acceptance_criteria>
- `backend/app/schemas/recommendation.py` exists (note: file path in task, should be services)
- `get_personalized_recommendations` returns movies not already fully watched
- `get_continue_watching` returns movies with progress < 100
- Returns empty list if no recommendations available
</acceptance_criteria>
</task>
