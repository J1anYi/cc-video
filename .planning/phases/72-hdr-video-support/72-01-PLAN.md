# Phase 72 Plan: HDR Video Support

**Phase:** 72
**Milestone:** v2.8 Advanced Media & Streaming Enhancements
**Created:** 2026-04-30

## Overview

Add HDR video detection, playback, and SDR fallback generation for optimal viewing on both HDR and non-HDR devices.

## Tasks

### Task 1: HDR Detection Service

**File:** `backend/app/services/hdr_service.py`

Create HDR detection and processing:
- Detect HDR10 metadata via FFprobe
- Parse color_transfer, color_primaries, color_space
- Store HDR metadata in VideoFile model
- Determine if source is HDR capable

**Validation:**
- Correctly identifies HDR10 content
- Stores metadata in database

### Task 2: VideoFile Model Extension

**File:** `backend/app/models/video_file.py`

Add HDR fields:
- is_hdr: boolean
- hdr_format: string (HDR10, HDR10+, etc.)
- color_space: string
- color_primaries: string
- color_transfer: string
- hdr_metadata: JSON

**Validation:**
- Migration applies successfully
- Fields populated on upload

### Task 3: Tone Mapping for SDR Fallback

**File:** `backend/app/services/transcoding_service.py`

Extend transcoding service:
- Add tone mapping filter for SDR generation
- Generate both HDR and SDR variants when source is HDR
- Use zscale or tonemap_opencl filter

**Validation:**
- SDR fallback generated from HDR source
- Quality acceptable

### Task 4: HDR Playlist Generation

**File:** `backend/app/services/hls_service.py`

Update HLS service for HDR:
- Add HDR tag to master playlist
- Include HDR variants with proper CODECS attribute
- Separate playlists for HDR and SDR

**Validation:**
- Master playlist includes HDR variants
- Players can select appropriate variant

### Task 5: Frontend HDR Detection

**File:** `frontend/src/components/VideoPlayer/HDRDetector.tsx`

Detect HDR capability:
- Check for HDR screen support
- Query display's color gamut
- Store user HDR preference

**Validation:**
- Correctly detects HDR-capable displays
- Preference persisted

### Task 6: HDR Playback Toggle

**File:** `frontend/src/routes/Playback.tsx`

Add HDR playback controls:
- HDR indicator badge
- Toggle between HDR/SDR
- Auto-select based on device capability

**Validation:**
- HDR badge shows for HDR content
- User can toggle HDR on/off

## Dependencies

- Task 1 → Task 2, Task 3
- Task 2 → Task 4
- Task 3 → Task 4
- Task 4 → Task 5, Task 6

## Testing Strategy

1. Unit tests for HDR detection
2. Integration tests for transcoding
3. E2E tests on HDR and non-HDR displays
4. Visual quality tests
