# Phase 71 Plan: Adaptive Bitrate Streaming

**Phase:** 71
**Milestone:** v2.8 Advanced Media & Streaming Enhancements
**Created:** 2026-04-30

## Overview

Implement HLS adaptive bitrate streaming with multiple quality variants and automatic quality selection based on network bandwidth.

## Tasks

### Task 1: Backend - Quality Variant Model

**File:** `backend/models/video_quality.py`

Create model for storing quality variants:
- VideoQuality model with fields: video_id, quality_name (4K/1080p/720p/480p), resolution, bitrate, playlist_path, segments_path
- Migration to add video_qualities table
- Relationship to Video model

**Validation:**
- Model created with proper relationships
- Migration applies successfully

### Task 2: Backend - Master Playlist Generator

**File:** `backend/services/hls_service.py`

Implement HLS master playlist generation:
- Generate master.m3u8 with all quality variants
- Include bandwidth and resolution metadata
- Support for audio group selection
- Endpoint to serve master playlist

**Validation:**
- Master playlist valid HLS format
- All variants properly referenced

### Task 3: Backend - Transcoding Pipeline Extension

**Files:** `backend/services/transcoding_service.py`, `backend/workers/transcode_worker.py`

Extend transcoding to generate multiple qualities:
- Detect source video resolution
- Generate variants: 4K (if source >= 4K), 1080p, 720p, 480p
- Use FFmpeg with hardware acceleration when available
- Store variant segments in organized directory structure

**Validation:**
- All variants generated for test video
- Segment durations consistent
- Transcoding completes without errors

### Task 4: Backend - Quality Variant Endpoints

**File:** `backend/routes/videos.py`

Add API endpoints:
- GET /api/videos/{id}/qualities - List available qualities
- POST /api/admin/videos/{id}/transcode - Trigger re-transcoding
- GET /api/videos/{id}/master.m3u8 - Get master playlist

**Validation:**
- Endpoints return correct data
- Admin-only routes properly protected

### Task 5: Frontend - Quality Selector Component

**File:** `frontend/src/components/VideoPlayer/QualitySelector.tsx`

Create quality selector UI:
- Dropdown with quality options (Auto, 4K, 1080p, 720p, 480p)
- Display current quality badge
- Auto mode indicator
- Persist selection preference

**Validation:**
- Selector displays all available qualities
- Selection updates player quality

### Task 6: Frontend - Adaptive Player Logic

**File:** `frontend/src/components/VideoPlayer/VideoPlayer.tsx`

Implement adaptive streaming:
- Use HLS.js for adaptive streaming
- Implement bandwidth estimation
- Auto-quality switching logic
- Smooth quality transitions
- Handle quality manual override

**Validation:**
- Player uses master playlist
- Auto-quality switches smoothly
- Manual override works

### Task 7: Frontend - Quality Stats Display

**File:** `frontend/src/components/VideoPlayer/QualityStats.tsx`

Display streaming statistics:
- Current bitrate
- Buffer health
- Dropped frames
- Quality switch events

**Validation:**
- Stats update in real-time
- Accurate representation of stream state

### Task 8: Admin - Transcoding Management UI

**File:** `frontend/src/pages/admin/TranscodingManager.tsx`

Create admin transcoding interface:
- View transcoding status per video
- Re-transcode specific qualities
- View transcoding history
- Cancel in-progress transcodes

**Validation:**
- Admin can view transcoding status
- Re-transcode triggers properly

5. Bandwidth simulation tests

## Risks

| Risk | Mitigation |
|------|------------|
| Transcoding time increases | Use hardware acceleration, queue jobs |
| Storage requirements grow | Implement cleanup for unused qualities |
| Bandwidth estimation inaccurate | Use HLS.js built-in ABR algorithm |
| Quality switching visible | Enable smooth switching with keyframe alignment |
