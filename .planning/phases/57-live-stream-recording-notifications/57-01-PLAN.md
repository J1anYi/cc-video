# Phase 57 Plan: Live Stream Recording & Notifications

**Phase:** 57
**Status:** Ready for Execution
**Created:** 2026-04-30

## Overview

Enable automatic recording of live streams and event notifications for users following live events.

## Requirements Coverage

| Requirement | Description | Tasks |
|-------------|-------------|-------|
| LIVE-06 | Live streams automatically recorded as VOD | 1-3 |
| LIVE-07 | Users receive notifications for followed events | 4-6 |

## Tasks

### Task 1: Add Recording Fields to LiveStream Model

**Goal:** Extend LiveStream model to support recording storage

**Files:**
- `backend/app/models/live_stream.py`
- `backend/app/database/migrations/`

**Implementation:**
1. Add recording_path field (nullable string)
2. Add recording_movie_id field (FK to Movie, nullable)
3. Add recording_status field (pending/processing/ready/failed)
4. Create database migration

**Verification:** Model accepts new fields, migration runs cleanly

---

### Task 2: Implement Segment Archival System

**Goal:** Archive HLS segments during live streaming

**Files:**
- `backend/app/services/recording_service.py`
- `backend/app/routes/live_stream.py`

**Implementation:**
1. Create RecordingService class
2. Implement segment archival during streaming
3. Store segments in recordings/{stream_id}/ directory
4. Track archived segment count

**Verification:** Segments are archived during streaming, directory structure correct

---

### Task 3: Create Recording Finalization on Stream End

**Goal:** Convert archived segments to VOD content

**Files:**
- `backend/app/services/recording_service.py`
- `backend/app/models/movie.py`
- `backend/app/routes/live_stream.py`

**Implementation:**
1. On stream end, generate VOD playlist from archived segments
2. Create Movie record with recording metadata
3. Link Movie to LiveStream via recording_movie_id
4. Update recording_status to ready

**Verification:** Ended streams create Movie records, VOD is playable

---

### Task 4: Create LiveEventFollow Model and API

**Goal:** Allow users to follow live events for notifications

**Files:**
- `backend/app/models/live_event_follow.py`
- `backend/app/routes/live_stream.py`

**Implementation:**
1. Create LiveEventFollow model (user_id, live_stream_id, notify_before_minutes)
2. Add POST /api/live/{stream_id}/follow endpoint
3. Add DELETE /api/live/{stream_id}/follow endpoint
4. Add GET /api/live/following endpoint for user followed events

**Verification:** Users can follow/unfollow events, follows are persisted

---

### Task 5: Implement Notification Dispatch System

**Goal:** Send notifications for upcoming and starting live events

**Files:**
- `backend/app/services/live_notification_service.py`
- `backend/app/tasks/notification_dispatcher.py`

**Implementation:**
1. Create LiveNotificationService
2. Add notification types: live_event_upcoming, live_event_starting, live_recording_available
3. Implement scheduled task to check upcoming events
4. Send notifications at 30 min, 5 min before scheduled start
5. Send notification when stream actually starts

**Verification:** Followers receive notifications at correct times

---

### Task 6: Add Frontend Follow UI and Recording Display

**Goal:** UI for following events and viewing recordings

**Files:**
- `frontend/src/pages/LiveEvents.tsx`
- `frontend/src/components/LiveEventCard.tsx`
- `frontend/src/pages/LiveRecordingPage.tsx`

**Implementation:**
1. Add Follow/Unfollow button to LiveEventCard
2. Show followed events indicator
3. Add recording badge to ended streams
4. Link to recording Movie page when available

**Verification:** Users can follow events, see followed status, access recordings

---

## Execution Order

1. Task 1 (Model extensions - no dependencies)
2. Task 2 (Recording archival - depends on Task 1)
3. Task 3 (VOD creation - depends on Task 2)
4. Task 4 (Follow model - no dependencies)
5. Task 5 (Notifications - depends on Task 4)
6. Task 6 (Frontend - depends on all backend tasks)
