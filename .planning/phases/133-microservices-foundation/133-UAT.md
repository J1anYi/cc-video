# UAT: Phase 133 - Microservices Foundation

## Test Cases

### 1. Service Client

#### TC-MS-001: Service Client Request
**Steps:**
1. Create ServiceClient instance
2. Make GET request to external service
3. Verify response

**Expected:** Request successful

#### TC-MS-002: Circuit Breaker
**Steps:**
1. Create client with circuit breaker
2. Simulate failures
3. Verify circuit opens after threshold

**Expected:** Circuit breaker opens

### 2. Service Registry

#### TC-MS-003: Register Service
**Steps:**
1. POST to /services/register
2. Verify service registered

**Expected:** Service registered

#### TC-MS-004: List Services
**Steps:**
1. GET /services
2. Verify registered services returned

**Expected:** Services listed

#### TC-MS-005: Service Health
**Steps:**
1. GET /services/{name}/health
2. Verify health status

**Expected:** Health status returned

### 3. Distributed Tracing

#### TC-MS-006: Request ID Propagation
**Steps:**
1. Send request with X-Request-ID header
2. Verify same ID in response

**Expected:** Request ID propagated

#### TC-MS-007: Trace ID Generation
**Steps:**
1. Send request without trace headers
2. Verify trace headers generated

**Expected:** Trace headers created

## Test Execution

```bash
# Register a service
curl -X POST http://localhost:8000/services/register \
  -H "Content-Type: application/json" \
  -d '{"name": "test-service", "url": "http://localhost:8001", "health_check_url": "http://localhost:8001/health"}'

# List services
curl http://localhost:8000/services

# Check service health
curl http://localhost:8000/services/test-service/health
```
