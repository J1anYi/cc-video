# UAT: Phase 135 - API Versioning

## Test Cases

### 1. Version Routing

#### TC-AV-001: Default Version
**Steps:**
1. GET /version without version header
2. Verify X-API-Version header in response

**Expected:** Version v1 returned

#### TC-AV-002: Header Version
**Steps:**
1. GET /version with X-API-Version: v2 header
2. Verify v2 in response

**Expected:** Version v2 returned

#### TC-AV-003: Path Version
**Steps:**
1. GET /v2/version
2. Verify v2 in response

**Expected:** Version v2 returned

### 2. Deprecation

#### TC-AV-004: Deprecate Endpoint
**Steps:**
1. POST /version/deprecate with endpoint info
2. GET /version/deprecated
3. Verify endpoint listed

**Expected:** Endpoint deprecated

#### TC-AV-005: Deprecation Headers
**Steps:**
1. Deprecate an endpoint
2. Call deprecated endpoint
3. Verify deprecation headers

**Expected:** X-API-Deprecated: true

### 3. Migration

#### TC-AV-006: Migration Guide
**Steps:**
1. GET /version/migration
2. Verify migration info returned

**Expected:** Migration guide returned

#### TC-AV-007: Changelog
**Steps:**
1. GET /version/changelog
2. Verify version history

**Expected:** Changelog returned

## Test Execution

```bash
# Get version info
curl http://localhost:8000/version

# Get with version header
curl -H "X-API-Version: v2" http://localhost:8000/version

# Deprecate endpoint
curl -X POST http://localhost:8000/version/deprecate \
  -H "Content-Type: application/json" \
  -d '{"method": "GET", "path": "/old-endpoint", "message": "Use /new-endpoint"}'

# Get migration guide
curl http://localhost:8000/version/migration
```
