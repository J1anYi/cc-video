# Phase 11-02: Password Reset Frontend

<objective>
Create frontend pages for password reset request and confirmation, integrating with the backend API.
</objective>

<read_first>
- frontend/src/routes/Login.tsx — Current login page structure
- frontend/src/routes/Register.tsx — Registration page pattern
- frontend/src/api/passwordReset.ts — Password reset API functions (created in 11-01)
- frontend/src/App.tsx — Route definitions
</read_first>

<tasks>

<task id="1" type="execute">
<name>Create ForgotPassword page</name>
<action>
Create `frontend/src/routes/ForgotPassword.tsx` with:
- Form with email input field
- Submit button that calls `requestPasswordReset(email)`
- Display success message after submission
- Link back to login page
- Use same styling patterns as Login.tsx
</action>
<acceptance_criteria>
- `frontend/src/routes/ForgotPassword.tsx` exists

- Calls API on form submission
- Shows success message after submission
</acceptance_criteria>
</task>

<task id="2" type="execute">
<name>Create ResetPassword page</name>
<action>
Create `frontend/src/routes/ResetPassword.tsx` with:
- Read token from URL query parameter (`?token=...`)
- Form with new password and confirm password fields
- Submit button that calls `confirmPasswordReset(token, newPassword)`
- Password validation (minimum length, matching passwords)
- Display success message and redirect to login
- Handle invalid/expired token error messages
- Use same styling patterns as Login.tsx
</action>
<acceptance_criteria>
- `frontend/src/routes/ResetPassword.tsx` exists
- Reads token from URL query parameter
- Has password and confirm password fields
- Validates passwords match
- Calls API on form submission
- Shows appropriate success/error messages
</acceptance_criteria>
</task>
