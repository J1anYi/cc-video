# Phase 11-01: Password Reset Backend

<objective>
Implement backend endpoints and services for password reset functionality, including token generation, email sending, and password update.
</objective>

<read_first>
- backend/app/routes/auth.py — Current auth routes structure
- backend/app/models/user.py — User model for reference
- backend/app/services/auth.py — Auth service patterns
- backend/app/config.py — Settings for SECRET_KEY, etc.
</read_first>

<threat_model>
- **Brute force attacks:** Rate limit reset requests per email (5 per hour)
- **Token enumeration:** Use cryptographically secure random tokens (32 bytes)
- **Token theft:** Tokens expire in 1 hour, single-use only
- **Email disclosure:** Don't reveal if email exists in system
</threat_model>

<tasks>

<task id="1" type="execute">
<name>Create PasswordReset model</name>
<action>
Create `backend/app/models/password_reset.py` with:
- `id: Mapped[int]` — primary key
- `user_id: Mapped[int]` — foreign key to users.id
- `token_hash: Mapped[str]` — SHA-256 hash of the reset token (not the token itself)
- `expires_at: Mapped[datetime]` — expiration timestamp
- `used_at: Mapped[datetime | None]` — when token was used (nullable)
- `created_at: Mapped[datetime]` — creation timestamp

Add relationship to User model in `backend/app/models/user.py`:
- `password_resets: Mapped[list["PasswordReset"]] = relationship("PasswordReset", back_populates="user")`
</action>
<acceptance_criteria>
- `backend/app/models/password_reset.py` exists and contains `class PasswordReset`
- Model has `user_id`, `token_hash`, `expires_at`, `used_at` fields
- `backend/app/models/user.py` contains `password_resets` relationship
</acceptance_criteria>
</task>

<task id="3" type="execute">
<name>Create email service</name>
<action>
Create `backend/app/services/email.py` with:
- `EmailService` class with `send_password_reset_email(email: str, reset_token: str, reset_url: str)` method
- Use Python's `smtplib` for email sending
- Read SMTP settings from environment variables:
  - `SMTP_HOST` (default: "localhost")
  - `SMTP_PORT` (default: 25)
  - `SMTP_USER` (optional)
  - `SMTP_PASSWORD` (optional)
  - `SMTP_FROM` (default: "noreply@example.com")
- For development, log email content instead of sending (when SMTP_HOST is not configured)
- Include reset link and expiration notice in email body
</action>
<acceptance_criteria>
- `backend/app/services/email.py` exists
- Contains `EmailService` class with `send_password_reset_email` method
- Reads SMTP config from environment variables

</acceptance_criteria>
</task>

<task id="4" type="execute">
<name>Create password reset service</name>
<action>
Create `backend/app/services/password_reset.py` with:
- `PasswordResetService` class
- `create_reset_token(db, user_id: int) -> str` — generates token, stores hash, returns plaintext token
  - Use `secrets.token_urlsafe(32)` for token generation
  - Hash with SHA-256 before storing
  - Set expiration to 1 hour from now
- `validate_token(db, token: str) -> User | None` — validates token, returns user if valid
  - Hash provided token and compare with stored hash
  - Check expiration and not already used
- `reset_password(db, token: str, new_password: str) -> bool` — validates token, updates password, marks token used
  - Use `auth_service.get_password_hash()` for new password
  - Set `used_at` to current time
- `cleanup_expired_tokens(db)` — removes expired tokens (optional cleanup task)
</action>
<acceptance_criteria>
- `backend/app/services/password_reset.py` exists
- Contains `PasswordResetService` with `create_reset_token`, `validate_token`, `reset_password` methods
- Tokens are hashed with SHA-256 before storage
- Token expiration is 1 hour
</acceptance_criteria>
</task>

<task id="5" type="execute">
<name>Add password reset endpoints</name>
<action>
Add to `backend/app/routes/auth.py`:

1. `POST /auth/password-reset` — Request password reset
   - Accept `PasswordResetRequest` (email)
   - Check if user exists by email
   - If exists, create reset token and send email
   - Always return success message (don't reveal if email exists)
   - Rate limit: 5 requests per email per hour (simple in-memory or use middleware)

2. `POST /auth/password-reset/confirm` — Confirm password reset
   - Accept `PasswordResetConfirm` (token, new_password)
   - Validate token and reset password
   - Return success or error message

Import required services and schemas at top of file.
</action>
<acceptance_criteria>
- `backend/app/routes/auth.py` contains `POST /auth/password-reset` endpoint
- `backend/app/routes/auth.py` contains `POST /auth/password-reset/confirm` endpoint
- Both endpoints work correctly with request/response schemas
</acceptance_criteria>
</task>
