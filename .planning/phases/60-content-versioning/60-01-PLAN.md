# Phase 60 Plan: Content Versioning

**Phase:** 60
**Status:** Ready for Execution
**Created:** 2026-04-30

## Overview

Support multiple versions per movie with separate metadata and watch history.

## Tasks

### Task 1: Create MovieVersion Model and Admin API

Goal: Allow admin to create and manage movie versions
Implementation:
1. Create MovieVersion model
2. Add version creation endpoint
3. Add version update/delete endpoints
4. Set is_default flag
5. Add version metadata fields
Verification: Admin can create versions with metadata

---

### Task 2: Add Version Selection to Frontend

Goal: Allow users to select version from movie detail page
Implementation:
1. Create VersionSelector component
2. Display available versions
3. Allow selection and play
4. Show default version indicator
Verification: User can select and play different versions

---

### Task 3: Implement Version-Specific Watch History

Goal: Track watch history per version
Implementation:
1. Add version_id to WatchHistory model
2. Update history recording
3. Fetch progress for specific version
4. Resume from correct version
Verification: History tracked separately per version

---

### Task 4: Add Admin Version Management UI

Goal: Admin interface for managing versions
Implementation:
1. Create MovieVersionManager component
2. List all versions for a movie
3. Add version creation form
4. Allow setting default version
Verification: Admin can manage versions through UI

---

## Execution Order

1. Task 1 (Model and API)
2. Task 3 (Watch history)
3. Task 2 (Frontend selection)
4. Task 4 (Admin UI)
