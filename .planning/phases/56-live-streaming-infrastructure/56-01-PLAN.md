# Phase 56 Plan: Live Streaming Infrastructure

**Phase:** 56
**Created:** 2026-04-30
**Status:** Ready for Execution

## Overview

Implement core live streaming capabilities for real-time video broadcasting.

## Tasks

### Task 1: Create LiveStream Model
**File:** `backend/app/models/live_stream.py`

Create SQLAlchemy model with:
- id, title, description
- scheduled_start, actual_start, end_time (DateTime)
- status: scheduled, live, ended
- stream_key (unique string)
- viewer_count (integer)
- admin_id (foreign key)

### Task 2: Add LiveStream API Routes
**File:** `backend/app/routes/live_stream.py`

Endpoints:
- POST /api/admin/live-streams - Create scheduled event
- PUT /api/admin/live-streams/{id} - Update event
- POST /api/admin/live-streams/{id}/start - Start stream
- POST /api/admin/live-streams/{id}/end - End stream
- GET /api/live-streams - List all (public)
- GET /api/live-streams/live - Currently live
- GET /api/live-streams/{id} - Event details

### Task 3: Viewer Count WebSocket
**File:** `backend/app/routes/websocket.py`

WebSocket endpoint for real-time viewer count:
- /ws/live-stream/{stream_id}
- Track connections per stream
- Broadcast viewer count updates

### Task 4: Frontend Live Events Page
**File:** `frontend/src/pages/LiveEvents.tsx`

- List upcoming and live events
- Show live indicator for active streams
- Link to watch page

### Task 5: Live Video Player
**File:** `frontend/src/components/LivePlayer.tsx`

- HLS.js integration for live playback
- Real-time viewer count display
- Connection status indicator

### Task 6: Admin Stream Management
**File:** `frontend/src/pages/admin/LiveStreamManagement.tsx`

- Schedule event form
- Start/End stream buttons
- Stream status display
